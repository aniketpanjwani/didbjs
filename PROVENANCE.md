# Provenance

This repository aims to make `didbjs` reproducible without blurring license
boundaries.

## Package License

`didbjs` package code is distributed under `MIT + file LICENSE`.

## Reference Roles

| Reference | License posture | Role |
| --- | --- | --- |
| Borusyak-Jaravel-Spiess Stata `did_imputation` | GPL-3.0 reference implementation | Behavioral oracle only. Expected outputs are generated by running licensed Stata; Stata implementation code is not copied or translated. |
| Announced Python `did_imputation` package | MIT-signaled package metadata | Public API/object-shape reference and parity target where features overlap. |
| Kyle Butts `didimputation` R package | MIT + license file | Existing R ergonomics and compatibility reference. |
| Algebraic/static smoke fixtures | Project-owned synthetic inputs and derived expected values | Default offline package checks and sanity verification. |

## Clean-Room Boundary

- Stata behavior may be inspected through help files, documentation, commands,
  generated logs, and expected output artifacts.
- Stata source code must not be copied, translated line-by-line, or used as the
  implementation structure for package code.
- MIT-licensed Python or Kyle material may be adapted only when the provenance
  ledger below records the artifact and required attribution.
- Reference regeneration scripts are test infrastructure, not runtime
  dependencies.

## Ledger

| Artifact | Source | Use | Attribution action |
| --- | --- | --- | --- |
| `R/` estimator and plotting code | Project implementation | Package runtime | Project MIT license |
| `tests/fixtures/smoke/f001-static-att/` | Synthetic fixture plus reference outputs | Shipped smoke fixture | Project ledger plus reference role notes |
| `tests/fixtures/parity/` | Synthetic fixtures and generated reference outputs | Full source-check parity suite | Excluded from source tarball; reference manifests record generator/source roles |
| `tools/parity/` | Project regeneration scripts | Developer validation | Not shipped in source package |
| Kyle compatibility tests/wrapper | Kyle R package behavior | Migration/compatibility surface | Preserve MIT attribution when code or examples are adapted |
| Python compatibility tests/wrapper | Python package behavior | API/object compatibility surface | Preserve MIT attribution when code or examples are adapted |

## Current Distribution Boundary

The source package produced by `R CMD build` includes package code, docs
generated by roxygen, smoke fixtures, API fixtures, installed spec files, and
check-result metadata. It excludes the heavyweight parity fixture directory and
developer regeneration tools.
